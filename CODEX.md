# Codex Adapter

Codex must read `AGENTS.md` first. This file is a thin adapter and is not the source of truth.

Codex rules for AI Logix:

- Follow `AGENTS.md` and the active sprint folder.
- Do not invent scope from conversation context.
- Work by sprint: requirements, blueprint, acceptance, handoff.
- Validate every change against `planning/sprints/current-sprint/acceptance.md`.
- For Sprint 000, use `planning/sprints/000-foundation-metrics/acceptance.md`.
- Report files created, files modified, validations run, skipped validations, and remaining risks.
- Update `planning/STATE.md` when phase status, validation status, or next phase changes.
- Update `planning/DECISIONS.md` when making durable technical or process decisions.
- Keep Phase 0 documentation-only unless a security/configuration cleanup is required.
